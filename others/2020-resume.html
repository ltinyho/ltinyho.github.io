项目经验

工作期间独立开发和主导了多个项目。

  • 2020.07-2020.08 UDP 通信协议设计 负责人

    背景: 为了提高车机和服务端通信的安全性,减少通信的流量,重新设计车机和服务端通信协议.

    职责:

    • 1、设计并实现通信协议

    • 2、兼容旧协议处理

      攻克的难点与业绩：

    • 1、增加通信安全性 参照原协议重新设计通信协议，增加基于 RSA 非对称加密协商 AES 对称加密密钥，后续通信采用 AES 对称加密，兼顾加密效率的同时提高通信的安全性。

    • 2、节省流量 将全数据传输改成多消息类型传输;将 json 字段名称缩减成单字符,通过文档来解释字段含义.整体流量节省至少 300%。
    • 3、兼容和扩展 为了兼容旧协议,增加适配层,新协议数据转换成旧协议数据格式;同时支持加密方式和序列化的扩展，只需实现对应接口即可。

  • 多渠道推送模块设计与开发

    背景： APP 需在多个国家提供服务，为了提高用户的触达率，接入多渠道推送模块,包括 FCM,APNS,极光,websocket 等。

    职责：

    • 1、接入多渠道推送

    • 2、实现 websocket 用户多设备连接和 websocket 多实例的改造

      攻克的难点与业绩:

    • 1、渠道接入高扩展性 定义推送接口，各渠道只需实现接口即可集成到推送模块。

    • 2、提高测试性 渠道差异较大,测试不便.通过增加单元测试,隔离对渠道环境的依赖,提高代码的健壮性,再没收到测试提交推送逻辑错误的 bug.
    • 3、为了实现用户多设备连接，基于用户 id 与设备 id 标识唯一连接，重构 websocket 模块。同时为了提高 websocket 的高可用和可扩展性，实现 websocket 服务多实例扩展，消除单点故障。

  • 换电柜项目

    背景: 换电柜是一款电动车无感换电产品,换电柜通过移动网络跟服务器通信,实现用户插入一块无电电池,弹出另一块可用电池,无感换电.

    职责：

    • 1、作为换电柜项目主负责人,负责项目产品的技术架构设计、通信协议制定、版本管理等工作。
    • 2、负责换电柜服务端整理框架设计和模块划分，以及服务端核心模块（iot、websocket 等）的编码实现。

    • 3、担任技术接口人，进行技术对接,参与技术方案的选型,方案的确定等等,推进换电柜的量产落地.

      攻克的难点与业绩:

    • 1、从 0 到 1 基于 k8s 搭建服务,负责技术架构设计、通信协议制定、版本管理等.模块划分采用分层架构,iot 设备数据控制层,负责设备状态上报和命令下发,上层业务层,解耦业务和基础服务.
    • 2、离线换电方案确定 为了提高用户换电体验，预期实现换电柜无网络离线换电，但单靠换电柜本身无法实现离线换电的逻辑，通过借助用户手机网络，实现换电，解决了无网换电的难题。
    • 3、项目推进和进度把控 协调内部前后端开发以及外部第三方硬件对接，跟产品、测试等团队密切配合，把控项目整体开发进度，合理规划工期，未出现延期情况。
    • 4、命令下发实时反馈

  • IM 客服系统

    为了满足市场营销需求，负责重构微信公众号客服系统。前端使用 vue、 element开发， 后端 go 连接 websocket 进行通信。编写分表脚本，将原有 6000w 数据表分表，同时优化查询语句,提高数据库查询效率,发送消息的响应时间从 3s 提升到 300ms。

  • 在线题库

    为了适应业务的快速变化，主导在线题库模块的重写。前端使用 vue 框架开发，重构答题系统，实现一套代码，适用手机端和电脑端，增强了业务的灵活性，减少了 30% 的开发工作量和维护成本。同时引入 typescript 强类型、 jest 进行单元测试，增强代码的可维护性，健壮性，提高了系统的稳定性。
    后端使用 go 开发，使用 redis 实现 mq 记录答题逻辑，将做题与统计解耦。
